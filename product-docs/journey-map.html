01
Use Case
Observation Capture
A supervisor or manager sees something in the field and logs it. The simplest and most frequent interaction. The entry point for the entire intelligence pipeline.
👤
Supervisor
Opens Capture — sees something on site
Taps Log Observation from home. Selects work type. Chooses Safe / At-risk / Near-miss. Writes what they saw. Submits.
Mobile · View 1
💡
If active visit plan exists, recommended topics surface as prompts — supervisor can reference them but is not locked to them.
⚙
System
Auto-tags and queues enrichment
Sets observer_id, worksite_id, timestamp, observer_role, cleared_for_sharing = true. Queues observation.enrich AI job.
Background · <1s
🤖
AI
Enrichment — classifies the observation
Reads free text. Infers failure_type (systemic / behavioural / environmental), key_hazard, anonymisation flags, confidence score. Stored in ai_suggested_* columns only.
Background · ~2s
⚙
Algorithm
Trend detection evaluates the observation pool
Counts near-misses by work_type + org_level in rolling window. Checks against configured thresholds. If threshold crossed → fires critical_insight.generate event.
Background · <1s
🔔 Notification Event — Conditional
⚠️
Threshold crossed — Critical Insight generating
Safety Manager (scope)
Push + Workbench inbox · "A pattern has been detected in [work type]. An insight is being drafted for your review."
➡️
If threshold not crossed: observation sits in the pool. No notification — routine capture should be silent to avoid noise.
02
Use Case
Trend Detection → Critical Insight
The algorithm detects a pattern. AI synthesises it into an actionable insight draft with a toolbox-ready narrative. Managers in the affected scope are invited to endorse before review.
⚙
Algorithm
Threshold crossed — insight generation triggered
N near-misses with same work_type in rolling window at an org level. Deterministic count — not AI. Creates CriticalInsight record with cleared_for_toolbox = false.
Background
🤖
AI
Drafts insight — pattern summary + veteran narrative
Receives anonymised observation cluster. Produces: pattern_summary, likely_systemic_cause, recommended_action, toolbox_narrative (veteran voice), escalation recommendation.
Background · ~8s
🔔 Notification Events — On Draft Complete
🔵
New insight ready for review
Safety Manager (scope)
Push + Workbench inbox · Urgent if 0 endorsements and >48h
💬
Field managers invited to endorse — "We've detected a pattern at your sites. Does this match what you're seeing?"
Managers (affected sites) Regional Manager (scope)
Push + In-app feed · Conversational tone — not a task, an invitation
💡
Managers can endorse with or without a comment. Each endorsement increments the count visible to the Safety Manager in their Workbench queue — before they open the item. High endorsement count = strong signal from the field.
👔
Manager / Regional Mgr
Reviews insight — endorses or comments
Opens insight from push notification. Reads AI draft narrative. Taps Endorse and optionally adds field context ("Saw this at my site Tuesday — spotter literally walked off mid-operation").
Mobile · Insight Thread
🔔 Notification Event
👍
New endorsement + comment on insight
Safety Manager Prior commenters
In-app feed · Digest (not immediate — batched per insight)
03
Use Case
Insight Review & Approval
The Safety Manager reviews the AI draft, reads field endorsements, makes a decision, and sets sharing scope. The most consequential human gate in the pipeline.
🛡
Safety Manager
Opens Workbench — insight appears in queue
Queue shows: insight type, work type, age, endorsement count (from managers). Context panel shows site metrics, event timeline, related items.
Desktop · Workbench
🛡
Safety Manager
Reviews AI draft + discussion thread
Reads source observations, AI pattern summary, toolbox narrative. Reads manager endorsements and comments. Decides: Approve / Edit / Reject. Sets sharing scope (site / region / division / org). Toggles systemic escalation if warranted.
Desktop · Work Panel
⚙
System — On Approval
cleared_for_toolbox = true · Insight enters content pool
Insight is now eligible for toolbox talk content selection at the approved scope. If escalate_to_systemic = true, systemic investigation workflow is triggered.
Background
🔔 Notification Events — On Approve
✅
Insight approved — visible in discussion thread
All managers who endorsed All commenters
Push + In-app feed · "The insight you endorsed has been approved and will reach crews in upcoming toolbox talks." Closes the loop for field participants.
📋
Talk content updated — insight now in pool for your site
Supervisors (in scope)
In-app feed only (not push) · "New safety insight is available for your next toolbox talk." Silent — next talk generation will include it automatically.
🔺
Systemic investigation initiated (if escalated)
Division Safety Manager Regional Manager
Push + Email · Escalation event logged in audit trail
💡
If rejected: Notify original AI generation system to reset cooldown for this work_type + level. No notification to field — rejected insights are silent failures, not public.

👤
Supervisor
Opens Talk view — talk ready badge on home
Taps the Talk card. Full-screen view opens. Sees AI-generated hazard intro, main narrative, key actions, discussion questions.
Mobile · View 2
👤
Supervisor
Optionally selects language + reads aloud
Selects crew language if non-English. AI translates. Taps Read Aloud — Web Speech API delivers talk. Supervisor can add a personal note (site-specific context AI can't know).
Mobile · View 2 · Kiosk
👤
Supervisor + Crew
Discussion questions — crew conversation
Supervisor uses the 3 AI-generated discussion questions. Crew responds. Supervisor notes anything new that surfaces (optional free text — feeds back as a new observation).
Mobile · In person
👤
Supervisor
Records crew attendance + delivers
Taps crew names from site list. Non-registered workers added manually. Taps Deliver — talk is locked. delivered_at set. Navigates to Record view.
Mobile · View 3 → 4

✅

🔔
Managers who endorsed the insight — loop closure notification
Endorsing managers
In-app feed · "The insight you endorsed has now reached [N] crew members across [M] sites." This is the moment the field manager sees their input had impact.

Sees portfolio sorted by atrophy score. Taps a high-atrophy site. Sees context: days since last observation, open investigations, near-miss trend, last visit date.

AI
Generates recommended observation topics

Mobile · Visit Detail
👔
Manager
Taps Start Visit — arrives on site, logs observations
Visit plan activated. Selected topics surface as prompts in the Capture form. Manager logs observations same as supervisor. observer_role = manager, visit_id linked automatically.
Mobile · View 1 (Capture)

Manager
Closes visit — summary generated

Mobile · View 4 (Record)

Incident → Investigation

Reports incident via Capture — "Did anyone get hurt?" → YES
Selects incident type (injury / near-miss / damage / environmental). Sets occurred_at. Describes what happened. Submits.
Mobile · View 1
⚙

Evaluates requires_investigation — creates Investigation record
Injury OR (near-miss + high-risk work type) → requires_investigation = true. Investigation record created with status = open. AI assistance job queued.

Use Case
Atrophy Alert
A site goes quiet — no observations for too long. The system detects it and routes an alert to the right people. The proactive intervention before silence becomes a problem.
⚙
Algorithm — Daily
Atrophy scores recalculated for all sites
Daily job. Calculates composite atrophy score per site. Sites crossing threshold (configurable, default 70) generate an Atrophy Alert queue item if no manager visit is already scheduled.
Background · Daily cron
🔔 Notification Events
📉
Site atrophy alert — [site] has gone quiet for [N] days
Assigned Manager Safety Manager
Push to manager · Workbench queue item for Safety Manager · "Riverside Tower hasn't had an observation in 18 days. A visit is recommended."
🛡
Safety Manager
Workbench — assigns manager to visit or dismisses
Atrophy Alert appears in queue. Safety manager can assign a manager to visit, or dismiss if site is on planned shutdown / known low-activity period.
Desktop · Workbench
🔔 Notification Event — On Assignment
🗓
You've been asked to visit [site] — atrophy alert
Assigned Manager
Push + In-app · Links directly to Visit Planning view for that site.
➡️
Assigned manager follows UC5 — Manager Field Visit. Atrophy score recalculates on visit completion. Alert clears from Workbench automatically.

Architecture
The Intelligence Pipeline
The Enquiry system is the pull counterpart to the Toolbox Talk push. Both are triggered by the same intelligence pipeline — one broadcasts learning to crews, the other gathers intelligence from sites. Three trigger sources, one reusable model.

Trigger 1
Critical Insight Approved
AI generates recommended question set. Safety Manager reviews, targets, dispatches. Default scope: source sites. Roles: supervisors.
Trigger 2
Investigation Mid-Enquiry
Investigator suspects cross-site condition. AI flags or investigator initiates. Blocked if legal_hold = true. Scope: same-region sites by default.
Trigger 3
Investigation Witness
Named individuals targeted. Person-to-person, not site-scope. Pre-populated from AI investigation framework. Sensitive notification wording.
09
Use Case
Investigation Mid-Enquiry
During an active investigation the AI assistant or investigator identifies a condition that may exist at other sites. An enquiry is dispatched to check — before the investigation closes, not after.
🤖
AI — Investigation Assist
AI flags a suspected cross-site condition
During framework completion, AI identifies that the root cause references a gap (procedure, equipment type, design) that likely exists across multiple sites — not just the incident site. AI adds a flag: "This condition may exist at other sites. Consider a mid-investigation enquiry."
Workbench · Investigation panel
💡
Investigator can also initiate manually without AI flagging — the option is always available in the investigation panel before close.
🛡
Investigator / Safety Manager
Legal hold check — is an enquiry safe to send?
System checks investigation.legal_hold. If true: enquiry creation is blocked entirely — no information gathering outside formal legal channels. If false: enquiry draft is created and AI generates questions from the investigation context.
System check · automatic
⚖️
Legal hold = hard block. The system never creates an enquiry from an investigation under legal hold. No override. This is enforced at the data layer.
🛡
Investigator
Reviews questions — sets scope and dispatches
AI-generated questions focused on verifying whether the specific gap exists at other sites. Investigator sets scope (default: same-region sites). Context narrative is partial — "we are investigating an incident at [site]" without investigation details.
Desktop · Enquiry Builder
🔔 Notification Event — On Dispatch
📋
Safety check from active investigation — your input needed
Supervisors (targeted sites)
Push + in-app · "We're currently investigating an incident at one of our sites. We need to check whether a related condition exists at your site. This will take about 3 minutes." — No investigation details disclosed.
👤
Supervisors
Respond to enquiry on mobile
Same response flow as a standard enquiry. Assurance checks, Work as Done, Gap Identification. Responses aggregate in real time in the investigation workbench panel.
Mobile · Enquiry view
🛡
Investigator
Enquiry results inform investigation findings
Mid-investigation enquiry results are visible inside the investigation panel — not as a separate report. The AI synthesis feeds directly into the investigation's contributing_factors and root_cause fields as additional evidence.
Desktop · Investigation panel
🔔 Notification Events — Results
📊
Enquiry results available — investigation EQ-[id]
Investigator Safety Manager
Push · "Responses from [N] sites confirm the condition exists beyond the incident site. See results in investigation EQ-[id]."
➡️
If results confirm the condition is widespread → scope of investigation broadens. If contained → investigation proceeds normally. Either way, the evidence base is stronger before the investigation closes.
10
Use Case
Investigation Witness Enquiry
Investigator identifies specific people — witnesses, participants, the affected worker's crew — and sends a structured question set to named individuals. Same model, person-targeted not site-targeted.
🛡
Investigator
Adds named individuals to the investigation
In the investigation panel, investigator selects named users from the platform directory — witnesses, supervisors on shift, crew members present. System creates witness enquiry draft with target_type = named_individuals.
Desktop · Investigation panel
⚖️
Legal hold check applies here too. Blocked entirely if investigation.legal_hold = true. When legal hold is lifted, the investigator can initiate then.
🤖
AI
Pre-populates witness question set from investigation framework
AI reads the investigation's current immediate_cause, contributing_factors context, and incident narrative. Generates witness-appropriate questions: Work as Done ("describe what you saw"), Assurance Check ("was the PTW signed and in place"), Gap Identification ("what would you change").
Background · ~8s
🛡
Investigator
Reviews, edits, dispatches to named individuals
Investigator reviews AI-drafted questions. Can add, remove, or reorder. No targeting needed — recipients are already named. Notification wording is specific and sensitive — unlike a standard site enquiry.
Desktop · Enquiry Builder
🔔 Notification Event — Witness Invitation
🔐
You have been identified as a witness — your account is needed
Named individuals
Push + Email · Sensitive wording: "You have been identified as a witness or participant in a safety investigation at [site]. Your input helps us understand what happened and prevent it happening again. Responses are confidential within the investigation team." — Name of incident not disclosed. No pressure language.
👤
Named Individuals
Respond on mobile — same enquiry flow
Identical mobile experience to a standard enquiry. Context narrative explains why they're being asked without disclosing investigation details. Work as Done responses capture what they actually witnessed. No character limits — more is better here.
Mobile · Enquiry view
🛡
Investigator
Responses feed investigation — not a standalone report
Witness responses appear in the investigation panel alongside the framework fields. AI synthesis extracts consistent themes across witness accounts. Investigator uses this to confirm or refine immediate cause and contributing factors before closing.
Desktop · Investigation panel
🔔 Notification Event
✅
Thank you — your response has been received
Respondent
In-app only · "Your input has been recorded. The investigation team will use it to understand what happened and make the site safer. You'll be notified of any relevant outcomes." — Closes the loop for the witness without disclosing investigation findings.
11
Use Case
Enquiry → Loop Closure
Enquiry closes. AI generates summary. Safety manager creates corrective actions. Summary feeds back into a Toolbox Talk — the field intelligence gathered by supervisors becomes the content delivered to crews. The loop closes completely.
🤖
AI
Summary generated on close
When enquiry closes (deadline reached or manually closed), AI generates the full learning summary: what was found, where the condition exists, what the field said in their own words, and recommended corrective actions. Three buttons immediately available: Create Corrective Actions, Generate Toolbox Talk, Export Summary.
Desktop · Enquiry Results
🛡
Safety Manager
Creates corrective actions from summary recommendations
One click from the summary to the corrective action creator. AI-recommended actions pre-populated with text, suggested owner type, and suggested due date. Safety Manager confirms or edits each. Same corrective_action entity — fully in the existing pipeline.
Desktop · Enquiry Results → Actions
🤖
AI
Generates Toolbox Talk from enquiry summary
The enquiry summary becomes a content source for a toolbox talk — a new trigger_source type. AI assembles a talk that reflects the field intelligence: "Here's what we asked across 4 sites. Here's what your fellow supervisors said. Here's what we're changing." This is the most credible toolbox talk content possible — it came from people doing the same work.
Desktop → Mobile delivery
🔔 Notification Events — Loop Closure
✅
Enquiry closed — here's what we learned
All respondents Endorsing managers
Push + feed · "The enquiry you contributed to is closed. Here's what we found across [N] sites. Your input shaped what happens next." — This is the notification that makes people respond to the next one. People engage when they see their input had an outcome.
📋
New talk ready — informed by field enquiry responses
All site supervisors (scope)
In-app feed · Same talk delivery flow. Content carries a new provenance marker: "Based on responses from [N] supervisors across [M] sites" — not just an AI insight, a genuine field consensus.
⟳
The loop is closed. Field signal → intelligence → enquiry → field intelligence → corrective action → toolbox talk → crews. Every person who contributed to the enquiry can see their input reached the people doing the work.
