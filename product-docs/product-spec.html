01Product Overview

Hiviz is a lean, opinionated safety intelligence system for construction and industrial organisations. It converts field observations and incident data into structured learning and crew communication — using AI to reduce the manual work between a field observation and a toolbox talk delivered to a crew.

The platform operates across two surfaces: a mobile app for field-facing roles (supervisors, managers, regional managers) and a desktop app for safety practitioners and divisional leadership. The two surfaces share a single data layer but serve fundamentally different workflows.

Design Principles

Core principle: Only the critical information required to drive operationally significant communication, learning, and improvement. Every field, every view, every prompt must justify its existence against this test.

The platform is built around four mandatory human touch points — the minimum viable human interactions that cannot be automated without compromising safety, legal integrity, or operational trust:

#Touch PointRoleWhat They DoWhy Human
T1CaptureSupervisor / ManagerSubmit observation or incidentA human sees the thing. Cannot be automated.
T2Insight ApprovalSafety ManagerReview and approve AI-drafted Critical InsightLegal and reputational gate before content reaches crews.
T3Investigation Sign-offInvestigator / Safety MgrClose investigation, set sharing eligibilityProfessional and legal liability requires human sign-off.
T4Talk DeliverySupervisor / ManagerReview generated talk, deliver to crewHuman context — knowing the crew and conditions — is irreplaceable.

Roles, Scope & Platform Access

Every role has a defined organisational scope. All data queries, content visibility, and workflow routing are scoped to the authenticated user's position in the org hierarchy. The platform enforces scope at the data layer — not the application layer.

Supervisor
Scope: Own worksite only
Mobile only
Capture (observations + incidents) · Talk review & delivery · Record · Site activity feed
Manager
Scope: Assigned worksites
Mobile primary · Desktop read
Visit planning · Capture (any assigned site) · Talk delivery (optional) · Site portfolio view
Regional Manager
Scope: All sites in region
Mobile primary · Desktop read
Identical to Manager — visit planning and capture scoped to region. Observations carry regional scope. Own observations feed regional trend detection.
Safety Manager
Scope: Division or region
Desktop primary · Mobile read
Workbench (insight approval, investigation sign-off, corrective actions, atrophy alerts) · Analytics · Reports · Admin
Division / Exec
Scope: Full division or org
Desktop · Report recipients
Analytics lenses · Board report · Export. Primarily receives generated reports. Not a daily platform user.
Regional Manager note: Regional managers do their own field visits and submit observations exactly as site managers do. The only differences are: (1) their scope covers all sites in the region, not just assigned sites; (2) their observations are tagged observer_role: regional_manager and carry a higher default sharing scope; (3) their portfolio view on mobile shows all regional sites ranked by atrophy. They are a first-class field role, not just a management layer.

02Mobile Views

The mobile app serves field-facing roles. All four mandatory human touch points occur here. The app is designed for one-handed use, works in bright sunlight, and functions with minimal connectivity (offline capture queued for sync).

View 1 — Capture
Mobile All field roles Touch point T1
UI Elements & Behaviour
  • Fork question: "Did anyone get hurt or could they have?" → YES routes to incident fields, NO routes to observation fields
  • Work type selector: Filtered to work types active at the user's current worksite. Required field.
  • Safety practice selector: Optional. Filtered to practices relevant to selected work type.
  • Observation type: Safe / At-risk / Near-miss — required for observation path
  • What was observed: Free text. Minimum 20 characters. No character limit.
  • Immediate action taken: Optional free text.
  • People count: Numeric stepper. Optional.

03Desktop Views

The desktop app serves safety managers and divisional leadership. It is a workbench and intelligence platform — not a field tool. The primary surface is the Workbench queue. Analytics and reporting are secondary surfaces accessed from the sidebar.

Workbench — Safety Manager Primary Surface
Desktop Safety Manager Touch points T2 + T3
Layout (4 columns)
  • Icon rail (56px): App switcher — Workbench, Analytics, Notifications, Investigations, Sites, Reports
  • Queue column (260px): Prioritised list of items requiring action. Filters: All / Insights / Investigations / Actions / Atrophy. Groups: Overdue / Due this week / Upcoming.
  • Work panel (flex): Selected item expands here. Full decision surface for the active item.
  • Context column (340px): Site/region context for the active item. Metrics, timeline, related items.
Queue Item Types
  • Critical Insight: AI draft awaiting approval. Shows pattern summary, source count, age. Actions: Approve / Edit / Reject.
  • Investigation: Open investigation needing sign-off or overdue. Shows incident ref, site, days open. Actions: Review fields, Close + set sharing.
  • Corrective Action: Overdue or approaching due date. Shows action, owner, days overdue. Actions: Mark complete, Chase owner, Extend.
  • Atrophy Alert: Site with atrophy score > threshold and no manager visit scheduled. Actions: Assign manager, Dismiss.
Scope Filtering
  • Queue always filtered to authenticated user's scope. A regional safety manager sees only items from their region.
  • Scope selector in topbar allows narrowing further (e.g. to a single site or work type).
Database Operations
  • Queue load: Union query across critical_insight (pending), investigation (open/overdue), corrective_action (overdue/due-soon), site_atrophy_view — all filtered to user scope, ordered by urgency
  • Insight approve: UPDATE critical_insight SET cleared_for_toolbox = true, reviewed_by_id = $user, reviewed_at = now(), review_action = 'approved', sharing_scope = $scope
  • Insight reject: Same UPDATE with review_action = 'rejected' — resets trend cooldown
  • Investigation close: UPDATE investigation SET status = 'closed', closed_by_id = $user, closed_at = now(), cleared_for_sharing = $bool, legal_hold = $bool, sharing_scope = $scope — triggers investigation.generate_narrative job if cleared
  • Corrective action update: UPDATE corrective_action SET status = $status, updated_at = now()
  • Context panel: Multiple reads — site metrics view, timeline events, related items JOIN
Key Queries
  • Queue: See View → Query Map section for full union query
  • Context timeline: SELECT observations, incidents, talks, visits WHERE site = $site ORDER BY occurred_at DESC LIMIT 20
Analytics — Intelligence Lenses
Desktop Safety Mgr + Division/Regional Mgr
Six Lenses
  • Intelligence Pipeline: Observation → Insight → Talk → Crew funnel. Volume, conversion rates, crew reached.
  • Leading Indicators: Near-miss rate, atrophy heatmap by site/week, failure type distribution (AI-classified), manager visit coverage.
  • Lagging Indicators: Incidents, TRIFR, LTI, near-miss:incident ratio — cross-referenced with observation volume and talk coverage per work type.
  • Site Health Map: All sites in scope. Atrophy score, last observation, last visit, open investigations, talk delivery rate.
  • Work Type Risk: Incidents + near-misses by work type. Trend lines. Talk coverage correlation.
  • Pipeline Health: Drop-off at each pipeline stage. Latency metrics. Corrective action close rate.
Global Controls
  • Scope selector: Org / Division / Region / Site. Cascades all charts.
  • Period selector: 7d / 30d / 90d / 12m
  • Role-based visibility: Safety manager sees all lenses. Division manager sees Leading, Lagging, Site Health only.
Key Queries per Lens
  • Pipeline funnel: COUNT observations, COUNT near-miss, COUNT critical_insight, COUNT critical_insight WHERE cleared, COUNT toolbox_talk WHERE delivered, SUM attendee_count — all for scope + period
  • Atrophy heatmap: COUNT observations GROUP BY worksite_id, week — LEFT JOIN all sites to show zero weeks
  • Failure type distribution: COUNT observations GROUP BY ai_failure_type WHERE ai_failure_type IS NOT NULL
  • Lagging by work type: COUNT incidents JOIN work_type, COUNT observations JOIN work_type, COUNT toolbox_talk JOIN work_type — GROUP BY work_type_id
  • Pipeline drop-off: Staged counts with latency AVG — see View → Query Map
Calculated Fields
  • Safety Culture Index: Composite of near-miss rate (40%) + talk delivery rate (40%) + manager visit coverage (20%). Score 0–100.
  • TRIFR: (Recordable injuries ÷ hours worked) × 1,000,000. Hours from payroll integration or manual input.
  • Near-miss:incident ratio: COUNT near-miss observations ÷ COUNT incidents for period.
Reports & Export
Desktop Safety Mgr + Exec
Report Types
  • Board Report: AI-generated narrative summary. Safety Culture Index, TRIFR, pipeline health, key actions. 1–2 pages. PDF export.
  • Manager Briefing: Per-manager summary. Sites visited, observations logged, insights generated, atrophy managed. For divisional review meetings.
  • Regulatory Package: OSHA/WHS-formatted incident register, investigation log with root causes and corrective actions, toolbox talk attendance records. Filtered by period and scope.
  • Raw Data Export: CSV of any entity set. Filtered by scope, period, work type, role. For third-party analysis.
  • Site Health Snapshot: Single-site PDF. Atrophy score history, all observations, talks delivered, open items. For manager briefings before site visits.
Database Operations
  • Board Report: Aggregation queries → AI narrative generation (Prompt 6 — Board narrative, not defined in earlier spec, new addition)
  • Regulatory Package: SELECT from incident JOIN investigation JOIN corrective_action JOIN toolbox_talk — scoped and period-filtered. No AI involved.
  • Raw export: Parameterised SELECT with scope and period filters. Sensitive fields stripped (observer names anonymised).
  • PDF generation: Server-side render. Client receives download URL.

View → Query Map

Every view that reads data is mapped to its primary queries here. This section is the primary reference for the backend developer.

Mobile: Manager Visit List — Atrophy Score Query

-- Atrophy score per site for a manager's scope
-- Returns sites ranked by composite atrophy score
WITH obs_age AS (
  SELECT worksite_id,
    EXTRACT(DAY FROM now() - MAX(observed_at)) AS days_since_obs
  FROM safety_intelligence.observation
  WHERE worksite_id = ANY($manager_site_ids)
  GROUP BY worksite_id
),
talk_age AS (
  SELECT worksite_id,
    EXTRACT(DAY FROM now() - MAX(delivered_at)) AS days_since_talk
  FROM safety_intelligence.toolbox_talk
  WHERE worksite_id = ANY($manager_site_ids)
    AND delivered_at IS NOT NULL
  GROUP BY worksite_id
),
open_inv AS (
  SELECT i.worksite_id, COUNT(*) AS open_count
  FROM safety_intelligence.investigation inv
  JOIN safety_intelligence.incident i ON inv.incident_id = i.id
  WHERE inv.status = 'open'
    AND i.worksite_id = ANY($manager_site_ids)
  GROUP BY i.worksite_id
),
nm_trend AS (
  SELECT worksite_id,
    COUNT(*) AS near_miss_30d
  FROM safety_intelligence.observation
  WHERE observation_type = 'near-miss'
    AND observed_at >= now() - INTERVAL '30 days'
    AND worksite_id = ANY($manager_site_ids)
  GROUP BY worksite_id
)
SELECT
  w.id, w.name, w.region_id,
  COALESCE(oa.days_since_obs, 999)  AS days_since_obs,
  COALESCE(ta.days_since_talk, 999) AS days_since_talk,
  COALESCE(oi.open_count, 0)        AS open_investigations,
  COALESCE(nm.near_miss_30d, 0)     AS near_miss_30d,
  -- Atrophy score: weighted composite 0–100
  LEAST(100, (
    COALESCE(oa.days_since_obs, 30) * 1.8 +
    COALESCE(ta.days_since_talk, 30) * 1.2 +
    COALESCE(oi.open_count, 0) * 8 +
    COALESCE(nm.near_miss_30d, 0) * 3
  )::INTEGER) AS atrophy_score
FROM public.worksite w
LEFT JOIN obs_age oa  ON w.id = oa.worksite_id
LEFT JOIN talk_age ta ON w.id = ta.worksite_id
LEFT JOIN open_inv oi ON w.id = oi.worksite_id
LEFT JOIN nm_trend nm ON w.id = nm.worksite_id
WHERE w.id = ANY($manager_site_ids)
ORDER BY atrophy_score DESC;

Desktop Workbench — Queue Union Query

-- Union all actionable items for safety manager's queue
-- Scoped to $user_scope_site_ids (derived from user's role + org hierarchy)
SELECT
  'insight' AS item_type,
  ci.id, ci.created_at AS triggered_at,
  ci.work_type_id, ci.generated_at_level, ci.pattern_summary AS title,
  EXTRACT(DAY FROM now() - ci.created_at) AS age_days,
  CASE WHEN EXTRACT(DAY FROM now() - ci.created_at) > 2
    THEN 'overdue' ELSE 'pending' END AS urgency
FROM safety_intelligence.critical_insight ci
WHERE ci.cleared_for_toolbox = false
  AND ci.review_action IS NULL
  AND ci.scope_ref_id = ANY($user_scope_ids)

UNION ALL

SELECT
  'investigation', inv.id, inv.opened_at,
  i.work_type_id, 'site',
  'INC-' || RIGHT(i.id::text, 4) || ' — ' || i.incident_type,
  EXTRACT(DAY FROM now() - inv.opened_at),
  CASE WHEN EXTRACT(DAY FROM now() - inv.opened_at) > $overdue_days
    THEN 'overdue' ELSE 'open' END
FROM safety_intelligence.investigation inv
JOIN safety_intelligence.incident i ON inv.incident_id = i.id
WHERE inv.status = 'open'
  AND i.worksite_id = ANY($user_scope_site_ids)

UNION ALL

SELECT
  'action', ca.id, ca.created_at,
  NULL, 'site', ca.action_text,
  EXTRACT(DAY FROM now() - ca.due_date),
  CASE WHEN ca.due_date < now() THEN 'overdue'
       WHEN ca.due_date < now() + INTERVAL '7 days' THEN 'due_soon'
       ELSE 'upcoming' END
FROM safety_intelligence.corrective_action ca
WHERE ca.status NOT IN ('completed', 'cancelled')
  AND ca.worksite_id = ANY($user_scope_site_ids)

ORDER BY
  CASE urgency WHEN 'overdue' THEN 1
              WHEN 'open' THEN 2
              WHEN 'due_soon' THEN 3
              ELSE 4 END,
  age_days DESC;

Scope & Filtering Rules

All data access is filtered by the authenticated user's organisational scope. Scope is resolved at query time from the users table and the org hierarchy. It is never stored on the request object — it is always derived fresh from auth.

RoleScope DerivationSite IDs QueryCan See Other Regions
Supervisor users.worksite_id directly WHERE id = user.worksite_id No
Manager users.assigned_worksite_ids[] (many-to-many) WHERE id = ANY(user.assigned_worksite_ids) No
Regional Manager users.region_id → all sites in that region WHERE region_id = user.region_id Own region only
Safety Manager users.scope_type + users.scope_ref_id → resolve all sites Recursive via org hierarchy to all sites in scope Within scope
Division Manager users.division_id → all regions → all sites Recursive via org hierarchy All in division
Implementation note: Create a PostgreSQL function get_user_site_ids(user_id UUID) that resolves the correct site list for any user based on their role and org position. Every scoped query calls this function. Never hardcode scope logic in application code.

05AI & Algorithm Pipeline

Five AI calls and four deterministic algorithms make up the intelligence pipeline. Algorithms own selection and gating (auditable). AI owns language, classification, and synthesis (inferential).

StepTypeTriggerInputOutputHuman Gate
1. Obs Enrich AI (Prompt 1) Observation created Free text + taxonomies failure_type, key_hazard, anon_flags None
2. Incident Triage Algorithm Incident created incident_type + work_type requires_investigation bool, investigation record None
3. Trend Detection Algorithm Observation created/enriched Observation pool + thresholds Threshold breach event None
4. Insight Draft AI (Prompt 2) Threshold breach Clustered observations (anon) pattern_summary, toolbox_narrative, escalation rec T2 — Safety Mgr
5. Inv. Assist AI (Prompt 3) Investigation created Incident narrative Suggested framework fields (ai_suggested_*) Investigator reviews
6. Inv. Narrative AI (Prompt 4) Investigation closed + cleared Confirmed framework fields toolbox_narrative, discussion_questions None post-close
7. Content Select Algorithm Talk generation requested Eligible content pool Ranked content set (max 3) None
8. Talk Assembly AI (Prompt 5) Talk generation requested Ranked content set Full structured talk JSON Supervisor reviews
9. Translation AI (Prompt raw) Language changed on talk view Full talk text + target language Translated plain text None
10. Visit Topics AI (Prompt 6) Visit plan detail opened Site observations, open inv., practice atrophy Recommended topic list with source badges Manager selects

Prompt Library Summary

Full prompt templates are in 02-ai-prompt-library.md. All prompts use claude-sonnet-4-20250514. All prompts output JSON except Prompt 9 (translation) which outputs plain text.

Algorithm: Atrophy Score

atrophy_score = LEAST(100,
  days_since_last_observation * 1.8  +  -- most weighted: observation culture
  days_since_last_talk        * 1.2  +  -- talk delivery cadence
  open_investigations_count   * 8    +  -- unresolved risk
  near_miss_30d_count         * 3       -- signal volume
)

-- Score bands:
-- 0–39:  Green  — healthy site
-- 40–69: Amber  — needs attention
-- 70+:   Red    — visit required
