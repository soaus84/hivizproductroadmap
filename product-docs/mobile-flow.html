03 — Key Design Decisions
01
One divergence point
The only difference between supervisor and manager mobile experience is the home screen. Supervisor sees today's site. Manager sees their site portfolio ranked by atrophy. Views 1–4 are identical code, different data.
02
Atrophy is a first-class signal
Score is calculated from: days since last observation, days since last talk, open overdue investigations, and near-miss trend vs site baseline. High atrophy = the system has gone quiet. That's always worth a visit.
03
Topics are recommended, never mandatory
AI and trend data surface recommended topics. Manager selects which to bring into the visit. They can add custom topics, deselect any, and the capture form uses selected topics as prompts — not locked fields.
04
Topic sources are transparent
Every recommended topic shows where it came from: a near-miss trend, an open investigation, an AI recommendation from practice atrophy, or a regional critical insight. The manager knows why each topic matters before they arrive.
05
Visit plan is a lightweight record
Starting a visit locks the plan — date, time, topics selected, site context snapshot. Post-visit, the record shows topics planned vs actually observed. Over time this creates a manager field presence audit trail.
06
Capture is the same form
Manager observations use the identical capture form as supervisors. The difference is auto-tagging: observer_role = manager, linked visit_id. This means manager observations feed the same trend detection and toolbox pipeline with no extra design cost.
